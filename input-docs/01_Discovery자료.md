## 킥오프 회의록 (요약)

**1. 모바일 청첩장은 어떻게 구성되나?**
- 모바일에서 접속하는 웹 페이지의 형태
- 자원은 GKS(GCP) 위에 구성하며, 추후 AKS(Azure) 환경으로도 옮길 수 있어야 함
- iOS와 안드로이드 환경 모두에서 동일하게 화면이 출력되어야 함(최소한 양식이나 비율이 깨지지 않아야함)

**2. 화면 구성은?**
- 모바일 웹에서 아래로 쭉 스크롤하면서 내용을 확인하는 방식
- 단순히 정적인 웹 페이지를 스크롤하는 것이 아닌, 스크롤을 내리면 다음 콘텐츠가 fade-in 됨
- 제일 상단(첫 화면)에는 메인 사진을 배경으로 두고, 제목(문구)를 그 위에 다른 레이어로 띄워놓음
- 순서는 메인 사진 + 제목 -> 본문(저희 결혼합니다. 잘 살겠습니다 메시지 + 신랑 신부 및 양가 부모님 정보) -> 신랑 소개 -> 신부 소개 -> 타임라인 -> 캘린더(2026년 11월 달력, 15일에 강조 표시, 결혼식 D-Day 카운터) -> 오시는 길(주소 + 지도 + 외부 앱 링크: 네이버지도, 카카오내비, 티맵 + 주차 안내) -> 참석 의사 전달(이름, 연락처, 참석여부, 인원, 식사여부) -> 갤러리(Blob Storage에 저장된 사진 갤러리 형식으로 출력) -> Instagram -> 마음 전하실 곳(신랑측, 신부측 계좌번호 & 카카오 페이)

**3. 사진은 어떻게?**
- 페이지에서 사용하는 이미지(사진)과 같은 정적 파일은, Blob Storage에 저장해서 불러온다
- 화면에서 Instagram 피드를 불러와서 출력하고, 해당 피드를 클릭하면 인스타그램 화면으로 넘어갈 수 있도록

**4. 동영상은?**
- 화면에서 Youtube 영상을 불러올 수 있으면 좋겠음, 접속 시 자동 재생될 수 있으면 재생되도록 하고, 안 되면 사용자가 재생 버튼을 누르도록

> **보류/미정 사항**
> - 동영상은 어떻게 활용할 것인가? Youtube 영상 불러오기는 확정은 아니지만, 업로드 방안만 확인해두자
> - 하객 사진첩(사진 or 동영상 업로드, Blob storage로 저장되도록) 포함 여부
