# 언어 표기 규칙 (한국어 기준)

모든 응답과 산출물은 **한국어를 기본**으로 작성한다. 영어는 아래에 해당하는 **기술 용어에 한해서만** 사용한다.

## 1. 기본 원칙

- 설명, 문장, 안내 문구는 한국어로 작성한다.
- 자연스러운 한국어 번역이 있어도 관용적으로 영어가 통용되는 기술 용어는 영어를 그대로 쓴다.
- 영어 용어를 무리하게 한글로 직역하지 않는다. (예: "의존성 주입" 대신 맥락에 따라 `Dependency Injection` 허용, 코드 식별자는 그대로 유지)

## 2. 영어로 표기하는 대상 (예외)

다음 항목은 번역하지 않고 영어/원문 그대로 표기한다.

- 코드 식별자: 클래스명, 메서드명, 변수명, 패키지명 등 (`BookService`, `useGeneratedKeys` 등)
- 파일 경로 / 파일명: `src\main\java`, `application.yml`, `build.gradle` 등
- 명령어 / 셸 명령: `.\gradlew.bat build`, `yarn dev` 등
- 라이브러리 / 프레임워크 / 도구 이름: Spring Boot, MyBatis, Flyway, React, Vite, AG Grid 등
- 표준 기술 용어 / 약어: REST, API, DTO, SQL, HTTP, JSON, ID 등
- 코드 블록 및 코드 주석은 해당 코드의 관례를 따른다.

## 3. 적용 예시

- ✅ "`BookService`의 조회 메서드를 추가하고 단위 테스트를 작성했습니다."
- ✅ "Flyway 마이그레이션으로 스키마를 관리합니다."
- ❌ "북 서비스의 조회 방법을 더했습니다." (코드 식별자/용어를 억지로 한글화)
- ❌ "Added a query method to BookService." (기본을 영어로 작성)

## 4. 적용 범위

- 채팅 응답
- AIDLC 산출물 문서 (요구사항, 설계, 계획 등)
- 커밋 메시지 / PR 설명의 본문 서술 (단, 형식 규약이 영어를 요구하면 그에 따른다)
